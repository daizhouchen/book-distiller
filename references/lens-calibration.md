# 视角自适应

## 核心原则

**不要求用户指定视角参数**。AI 自己判断。

## 判断逻辑

### 默认：balanced（全貌）

绝大多数情况下，产出 **balanced**——全貌视角，不偏重任何一个维度。

### 触发场景 1：对话上下文显示用户领域偏好

如果对话历史显示用户：
- 最近聊过**商业/创业/战略**话题 → 给商业视角加权（不牺牲全貌）
- 最近聊过**心理学/咨询/亲密关系** → 给心理视角加权
- 最近聊过**编程/技术/系统设计** → 给技术视角加权
- 最近聊过**文学/写作/美学** → 给文学视角加权

**加权 ≠ 偏废**：基础全貌仍在，只是相关视角的内容更展开。

### 触发场景 2：用户明确要求

如果用户说：
- "我想从商战角度看遥远的救世主"
- "帮我分析金瓶梅里的女性心理"
- "从系统设计角度读代码整洁之道"

→ 切到对应 lens 模式：
- `business` / `psychology` / `literary` / `technical` / `historical` / `philosophical`

### 触发场景 3：书的基因天然导向某视角

有些书天然强势一个维度，不需要额外加权：
- 《遥远的救世主》 → 商战 + 哲思
- 《红楼梦》 → 文学 + 家族社会学
- 《原则》 → 方法论 + 投资
- 《代码整洁之道》 → 技术

Skill 的**默认 balanced 其实已经考虑这些**，不需要用户操心。

## Lens 透明标注

每个 lens 切换在产出开头**明确说明**：

```markdown
【本次产出视角】商业视角 📈

  为什么选这个视角：
  近期对话显示你在思考公司战略问题，本报告在保持全貌
  的前提下，对"决策链""信号博弈""组织激励"等维度
  做了展开。如果你希望看到全貌版或切换到心理/文学视角，
  告诉我即可。
```

## Lens 参数清单

| Lens | 触发关键词 | 加权的内容 |
|---|---|---|
| `balanced` | （默认） | 全貌，不加权 |
| `business` | 商战、创业、战略、管理、投资 | 决策链、博弈、组织、商业模式 |
| `psychology` | 心理、情感、关系、动机、成长 | 人物动机、防御机制、情感动力 |
| `literary` | 文学、写作、美学、语言 | 叙事结构、语言肌理、象征系统 |
| `technical` | 编程、技术、系统、架构 | 方法论、模型结构、边界条件 |
| `historical` | 历史、时代、社会 | 史料交织、时代背景、结构性归因 |
| `philosophical` | 哲学、存在、意义 | 论证深度、形上学含义 |

## 实现

`distill.json` 的 `meta.lens` 字段记录当次视角：

```json
{
  "meta": {
    "lens": "business",
    "lens_reason": "用户近期讨论创业决策，加权商业维度但保持全貌"
  }
}
```

`render.py` 根据 lens 调整内容权重（不删除其他维度，只调整展开详尽度）。

## 反模式

### 反模式 1 · 盲目加权

没看上下文就随便加权 → 用户看了觉得偏。**规则**：默认 balanced，除非有强信号。

### 反模式 2 · 偷懒跳过标注

产出的是商业 lens 却不告诉用户 → 信任受损。**规则**：任何非 balanced 都要标注。

### 反模式 3 · 过度切换

每次产出都切换一个 lens → 读者看不到全貌。**规则**：如果用户想要多视角，可以产出两份。
