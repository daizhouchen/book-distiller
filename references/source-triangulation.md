# 信源分级与交叉验证

## 分级表

| 级 | 来源 | 可信权重 | 使用原则 |
|---|---|---|---|
| **A** | 原书原文 / 权威译本（商务、中华、上海译文、企鹅经典等） | 1.0 | 唯一金标准；所有观点必有 A 级锚点 |
| **B** | SSCI/CSSCI 论文、权威学术专著、作者本人访谈、知名学者署名长文、豆瓣 1000+ 赞研究型长评 | 0.7 | 可作为角度启发；不冲突时引用 |
| **C** | 普通书评人、知乎高赞、公众号、豆瓣短评、普通媒体评论 | 0.3 | 仅作参考对照；不能独自支持观点 |
| **D** | 模型自身知识（未核证） | 0.2 | 补全用；必须标注"未核证" |

## 交叉验证规则

### 规则 1 · 唯一金标准原则

**任何重大观点必有 A 级锚点**。没有 A 级支持 → 观点不成立，必须改或砍。

### 规则 2 · A 级内部矛盾处理

如果原书不同章节说法矛盾，**并列呈现，不强行统一**。
- 示例：金瓶梅前后对西门庆的态度有分裂（前半同情多，后半警世强），这是作者本意，不要抹平。

### 规则 3 · B/C 级与 A 级冲突

以 A 级为准。B/C 级的观点降为"存疑视角"或"一家之言"，可以保留但要标注。

### 规则 4 · 只有 C/D 级支持

观点降格为"延伸思考"——放在"思想坐标"或"思考工具箱"，不进主正文。

### 规则 5 · 透明标注

**产出开头必须明确标注信源构成比例**：

```
【信源构成】
  A 级（原书原文）：45%
  B 级（学术/权威解读）：30%
  C 级（普通书评/网络）：20%
  D 级（模型知识补全）：5%

  置信度自评：高 ⭐⭐⭐⭐
```

置信度标准：
- **高 ⭐⭐⭐⭐⭐**：A ≥ 60% + B ≥ 20%
- **中高 ⭐⭐⭐⭐**：A ≥ 40% + A+B ≥ 65%
- **中 ⭐⭐⭐**：A ≥ 20% + A+B ≥ 50%
- **低 ⭐⭐**：A < 20%，警示"基于二手资料"
- **不宜产出 ⭐**：A = 0 且 B < 30%（建议用户上传原文）

## 采集路径

### 优先级 1 · 原书文本

**公版书**（作者死后 ≥ 50 年）：
- ctext.org（中国哲学书电子化计划，古籍权威）
- 古诗文网（gushiwen.org，中文古籍）
- Wikisource 中文（zh.wikisource.org）
- archive.org（英文公版）
- Project Gutenberg（gutenberg.org，英文公版）

**现代书**：
- 合法的电子书资源、豆瓣阅读、微信读书、Kindle
- 不碰 paywall，不绕 DRM

**实操**：
```python
# 示例调用
from scripts.fetch_sources import try_fetch_original
text, level = try_fetch_original(book_title, author)
# level = "A" 表示成功
# level = None 表示失败
```

### 优先级 2 · 用户上传

原书抓不到，**明确请求用户**：

> "我没能从公开渠道获取到《XXX》的原文。如果你方便，请把你手上的电子版（PDF/TXT/epub）路径告诉我。"

### 优先级 3 · 二手资料拼凑

用户没有 → 启动二手资料模式：

**来源**：
- 权威梗概（Wikipedia、豆瓣书页、百度百科的"内容简介"）
- 学术论文摘要（Google Scholar 检索"书名 + 综述/批评"）
- 豆瓣长评 TOP 10（按点赞数排序，取有实质分析的）
- 知乎高赞（"如何评价 XXX"问题下）
- 作者访谈（YouTube/B 站/媒体报道）

**必做**：
- 至少 3 个信源交叉
- 矛盾点单独标注
- 产出开头警示"**本报告基于二手资料，未直接读原文，置信度为中**"

### 副线（始终执行） · 他山之石采集

无论是否有原文，都抓 3-5 份**权威解读**作为对照：

- 知名学者的评论（刘再复/夏志清/李泽厚类）
- 国外汉学家的解读（宇文所安/浦安迪类——对中文古籍）
- 严肃书评媒体（《读书》《书城》/ NY Review of Books / LRB）
- 百家讲坛/公开课录音转文字

这些不是 A 级证据，是**他山之石**——提供不同角度，我们自己做采纳/存疑判断。

## 他山之石的使用

### 错误做法（单独成章罗列）

> ## 他山之石
> 某学者认为……
> 另一学者认为……
> 还有学者认为……

读起来像 paper 的 literature review，冷漠无聊。

### 正确做法（融入各模块做对照）

在每个模块的主干分析完成后，附一段**短对照**：

```markdown
【主分析】
丁元英的"文化属性"论，其实可以从冷兵器时代的"道-术"之辨来理解……

【对照】
  评论者 XX 在《豆瓣长评》中提出这个"文化属性"
  更接近梁漱溟"理性早启"的判断。
  我们部分认同——两者确有家族相似。但豆豆原作
  更强调"强势/弱势"的对抗面，这一层在梁的论述里
  不那么突出。所以两者可互相补充，不能相互替代。
```

这种融入式对照让外部视角**参与论证**，而不是旁白陈列。

## 标注规范

正文中引用外部来源，统一格式：

```
（来源：刘再复《双典批判》p.123）
（来源：豆瓣长评「欲望的经济学」2019，@用户名）
（来源：作者访谈《凤凰卫视》2008-03）
```

模型补全的观点，明确标注：
```
（观点来源：本分析基于文本结构推导，未核证作者是否亲证）
```

## 失败模式

### 失败 1 · C 级当 A 级用

豆瓣短评说"西门庆代表明代商人精神"——这是 C 级观点，不能直接当结论，必须**回原书找证据**才能决定是否接受。

### 失败 2 · 信源不透明

通篇不标引用来源，读者无法验证。**规则**：每个非原文的关键观点，括注来源。

### 失败 3 · 用二手资料偷懒

能拿到原文却因为懒而用梗概——**绝对禁止**。二手资料只在**拿不到原文**时使用。
