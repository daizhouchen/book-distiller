# 框架内化：二阶段写作法

## 问题背景

读书笔记最容易翻车的地方：**框架硬贴脸上**。

### 反例（让人出戏）

> 从马斯洛需求层次理论来看，西门庆对女色的占有属于生理需求和安全需求的扭曲满足。从社会交换理论视角，他与蔡京的关系是一种典型的资源交换。结合布迪厄的场域理论，他在官场中处于弱势位置……

**问题**：框架名满天飞，读者被术语搞累。而且——**用框架名充当论证本身是偷懒**，真正的论证是"为什么这样？"。

### 正例（内化的样子）

> 西门庆对女色的贪婪，表层是性，深层是对失控感的镇压——他出身破落户，幼年父亲缺席，他需要通过占有来证明自己"能"。这种占有是扭曲的：当一个人把性当成安全证明，他对对方就不再有好奇。所以你看，金莲、瓶儿、惠莲，他从未问过她们一句真正的问题。

同样用了需求层次思想 + 依恋理论 + 防御机制，但**没有点名一个框架**。读者只感受到思考的深度，不感受到术语的压迫。

## 二阶段写作法

### 第一阶段 · 草稿（含框架名）

**允许自由使用框架名**。目的：让思考走完整。

做法：
1. 边写边用框架辅助思考
2. 可以直接写"这里从 X 理论看……"
3. 只是草稿，不追求可读性

这一阶段产出一个"思维透明"版——可以看到你用了哪些工具。

### 第二阶段 · 成稿（去框架名）

**把正文里所有框架名删掉**，只保留框架导出的洞察和论证。

做法：
1. 检索正文里所有术语名（马斯洛/弗洛伊德/博弈论/SWOT/场域……）
2. 把包含术语名的句子重写——**用自然语言复述同样的思考**
3. 确保论证强度不降低

重写示例：

**草稿**："从博弈论看，丁元英在这里选择了一个混合策略。"

**成稿**："丁元英没有完全投入，也没有完全撤出——他留了一条可以反悔的退路。这让对手无法锁定他的下一步，也让他自己的损失可控。"

后者没有"博弈论"三个字，但思考深度一样。

### 自检问题

删掉框架名后问自己：
- 论证强度有没有下降？
- 读者能 follow 推理吗？
- 如果还是"因为 X 理论所以 Y"的结构，说明没真正内化——框架思想**应该长在文字肌理里**，不是挂在句首

答"是" → 说明框架真的内化了。

### 重要区分：框架名 vs 历史人物名

**禁用的是抽象框架名**：
- ✗ "马斯洛需求层次理论"、"博弈论"、"场域理论"、"囚徒困境"、"边际效用"
- ✗ "三幕结构"、"英雄之旅"、"陌生化"

**允许的是历史人物姓名**（作为承继关系、文化坐标的必要标识）：
- ✓ "勒庞承继自塔尔德" —— 谈思想史绕不过人名
- ✓ "弗洛伊德后续发展了类似观察" —— 讲影响链需要人名
- ✓ "韦伯在同时期提出过相似问题" —— 历史对勘

**判别原则**：
- 如果一个名词是**思考工具的名字**（学派/理论/概念）→ 禁用
- 如果是**历史上的人物**（人们知道他做了什么）→ 允许

人名即使同时是框架名（如"马斯洛"），**作为人名引述时可用**，但其理论的抽象名（"马斯洛需求层次"）仍禁。

### 唯一例外

文末「**思考工具箱**」折叠区可以低调列出本文思考时用过的框架名：

```
【思考工具箱】（折叠）
  · 马斯洛需求层次 —— 用于分析西门庆对安全感的扭曲追求
  · 依恋理论 —— 理解他为什么无法与任何女性建立深度关系
  · 交易成本理论 —— 解释他买官的经济理性
  · ……
```

这是正文**唯一**出现框架名的地方，给想深挖的读者延展线索。

## 实战演练

### 场景：分析《遥远的救世主》中丁元英的决策

**第一阶段草稿**：

> 丁元英在启动格律诗项目时运用了**博弈论**的信号理论——他给王庙村的村民发出"无偿指导"的信号，但**信息不对称**的真相是他完全知道这个项目会带来剧烈冲突。他选择**混合策略**——半投入半观望，为自己留了**退出期权**。从**交易成本**角度，他绕过了传统的企业架构，直接组织村民生产，降低了组织成本。

**第二阶段成稿**：

> 丁元英启动格律诗时表面说的是"无偿指导"，其实他心里清楚这场仗会撕裂所有相关方的关系——他没说的比说的多。他没把自己完全绑进去：公司架构刻意留了一些松动，万一局势崩坏他能抽身。同时他跳过了"建公司、聘经理、定工资"这套标准戏路，直接让村民变成生产者，相当于用一根绳子把所有环节串起来——绳子短，损耗就小。这不是精明，是冷峻。他知道每一个动作都在算账。

**对比效果**：
- 草稿：术语 5 个，读者疲惫
- 成稿：术语 0 个，但思考深度一致，且多了文学质感（"他没说的比说的多"、"冷峻"、"算账"）

## 规则总结

1. **自由用框架想，严格管框架写**
2. **正文禁用框架名**（工具箱除外）
3. **删除术语后论证强度不能降**——能降说明术语是拐杖
4. **框架应该长在文字肌理里**——不是挂在句首
5. **工具箱折叠呈现**——尊重不同读者的兴趣深度
