# 核心主旨详述

## 四句宪法

```
1. 不能空有观点
2. 事实支撑观点
3. 揭示系统逻辑
4. 事实陈述本身也要系统化
```

这四句构成整个 Skill 的底层逻辑。违反任一条，产出都算不合格。

## 为什么这四条这么重要

### 第一条：不能空有观点

大多数读书笔记的失败第一名：通篇都是观点，没有事实。

**反例**：
> 西门庆是一个极度贪婪的人，他对权力、金钱、女色都有无止境的欲望，这体现了明代商人阶层的精神危机。

这段话有三个观点叠加，读者没法判断对错，拿不走任何东西。

**正例**：
> 西门庆在第十八回拿出 500 两银子贿赂蔡太师府门子（这是当时一个七品官两年的俸禄），只为让蔡京写一张人情条子。这不是个别行为：全书类似桥段出现 11 次，覆盖从地方官到京官的全链条。由此看出——**他对权力的兴趣不在做官，而在用钱买到能不被官整治的安全**。

事实（500 两、11 次、覆盖全链条）→ 推论（关心的不是做官）→ 结论（是买安全）。链条清晰可驳。

### 第二条：事实支撑观点

不是事实堆砌，而是**显式的事实→推导→观点链条**。

**规范做法**：
```
【事实】[具体情节/数据/引用 + 章节定位]
  ↓
【推导】[3-5 步显式逻辑]
  ↓
【观点】[结论，且结论范围不能超出推导能支持的范围]
```

每个核心观点必须能追回至少一个 A 级事实锚点。

### 第三条：揭示系统逻辑

碎片洞察是零分产出。**系统逻辑**是：

- 这本书的底层运作机制是什么？
- 作者用什么"看不见的结构"串起所有表层内容？
- 这个结构一旦被识别，全书的枝节都能各归其位。

**例子**：
- 金瓶梅的系统 = **欲望的自噬三角循环**：情欲→消费→反哺钱欲→购买权力→权力又放大情欲。
- 遥远的救世主的系统 = **文化属性的"硬度"链**：强势文化(丁) vs 弱势文化(叶/村民)的对抗与渗透。
- 乌合之众的系统 = **群体心理的去个性化三阶**：情绪传染→行为传染→身份认同传染。
- 非暴力沟通的系统 = **四步闭环**：观察→感受→需要→请求。每一步的"堵塞"对应一种沟通失效。

**没识别出系统 = 没读懂**。

### 第四条：事实陈述本身也要系统化

这是 v5 升级的核心洞察。

事实堆在一起还是散沙。必须经过四步处理才能支撑观点：

1. **穷尽式提取**（不挑只收）
2. **维度分类**（进坐标系）
3. **模式识别**（找单维规律）
4. **跨维度交织**（看系统浮现）

详见 `fact-systematization.md`。

## 反面案例对照（什么是不合格产出）

### 案例 A · 全是观点（违反第一条）

> 乌合之众深刻揭示了群体心理的规律。勒庞指出，人在群体中会丧失个性，变得非理性。这对我们今天的社交媒体有重要启发。

**诊断**：三句话三个观点，没有一个事实锚点。不合格。

### 案例 B · 事实堆砌（违反第二条）

> 西门庆在第一回出场。他有六个妻妾。他是山东清河县人。他和王婆勾结。他做布料生意。他后来得到蔡京的提拔。

**诊断**：全是孤立事实，没有推导链，没有结论。不合格。

### 案例 C · 碎片洞察（违反第三条）

> 丁元英精通佛教。他擅长投资。他理解弱势文化。他知道怎么利用格律诗的生产成本差异。

**诊断**：每一点单独看都对，但没有揭示"这些点如何构成一个整体决策系统"。读者看完不知道怎么学。不合格。

### 案例 D · 事实零散（违反第四条）

> 金瓶梅里有很多性描写。作者关心女性命运。李瓶儿最终病死。潘金莲谋害亲夫。西门庆突然暴毙。所以这本书很悲剧。

**诊断**：事实都是对的，但**没有按维度组织**，也**没有识别事实之间的关系**。读者看不到系统。不合格。

## 合格产出的基本形态

任何一个模块都应该长这样：

```
╔═══════════════════════════════════════════════════════════╗
║  【事实层】                                                 ║
║                                                            ║
║   这里呈现系统化的事实：                                      ║
║   - 按维度 A 分类的事实 X 条                                  ║
║   - 按维度 B 分类的事实 Y 条                                  ║
║   - 可视化为矩阵/树/轴                                        ║
║   - 每个事实都有章节锚点                                      ║
║                                                            ║
╠═══════════════════════════════════════════════════════════╣
║  【机制层】                                                 ║
║                                                            ║
║   从事实系统中提取：                                          ║
║   - 单维度规律：X 维度呈现递减/递增/循环 pattern              ║
║   - 跨维度交织：A×B 形成互相转化的链条/三角/层级              ║
║   - 底层机制：为什么会这样？                                   ║
║                                                            ║
╠═══════════════════════════════════════════════════════════╣
║  【观点层】                                                 ║
║                                                            ║
║   基于机制得出：                                              ║
║   - 核心判断：作者在说什么                                     ║
║   - 现代迁移：这对今天什么启示                                  ║
║   - 边界：哪些结论不能推广                                     ║
║                                                            ║
╚═══════════════════════════════════════════════════════════╝
```

## 自检问题（每写完一个模块问自己）

1. 这个模块有没有事实锚点？至少几个？
2. 事实是系统化的还是罗列的？用什么坐标系？
3. 推导链清楚吗？一个读者能 follow 每一步吗？
4. 观点的范围有没有超出事实能支持的范围？
5. 这个模块揭示了系统逻辑的哪一块？还是只是孤立洞察？
6. 如果删掉观点层只留事实层，读者能自己走到相似的结论吗？（好的事实层会引导，差的要靠观点兜底）

答"不确定"或"没有"的，回改。
