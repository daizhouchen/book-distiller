# 骨架 · 模型基因主导

适用于给出可操作方法/工具的书：非暴力沟通、认知觉醒、原则（兼）、思考快与慢（兼）、代码整洁之道、设计模式、GTD、精益创业等。

## 模块顺序

1. 卷首 · 开篇 + 母题 + 记忆抓手
2. 模型全图（SVG）
3. 模型分步详解
4. 适用情境 × 失效情境（矩阵）
5. 实操剧本
6. 模型背后的系统逻辑
7. 误读陷阱
8. 作者/模型的盲点
9. 金句精选
10. 思想坐标
11. 思考工具箱（折叠）

## 各模块内容要求

### 1 · 卷首

同通用规范。

### 2 · 模型全图（SVG）

- 完整的模型流程/结构图
- 每一步有图标/节点
- 节点间的箭头显示输入输出
- 关键分叉点朱砂标注

例：非暴力沟通四步图
```
  观察 → 感受 → 需要 → 请求
   ↑      ↑      ↑      ↑
  (去评价)(去解释)(去脚本)(去命令)
```

### 3 · 模型分步详解

每一步按**三重结构**：

**事实层**（步骤的具体构成）：
- 这一步是什么
- 原文怎么定义的
- 作者给的例子
- 事实矩阵（步骤要素 × 操作细节）

**机制层**（为什么有效）：
- 这一步处理了什么心智动作
- 如果跳过会发生什么
- 与前后步骤如何联动

**观点层**（什么时候用 / 怎么用）：
- 日常场景落地
- 陷阱提示
- 熟练化路径

**原文对照**：作者给的操作范例 × 解读
**外部视角**：实践者的反馈/批评

### 4 · 适用情境 × 失效情境

⭐ 模型型书的专属系统化：

- 矩阵 SVG：场景类型 × 模型适用度
- 显示**什么时候模型好用，什么时候不好用**
- 失效情境要具体，不能泛泛

例：非暴力沟通的适用/失效
```
              权力对等    权力失衡    结构性暴力
对话场景      ✓ 好用      △ 有限      ✗ 不适用
辩论/谈判     ✓ 好用      △ 要补充    ✗ 不适用
PUA/霸凌      ✗ 不适用    ✗ 不适用    ✗ 需要退出
```

### 5 · 实操剧本

- 给 2-3 个具体剧本
- 每个剧本：情境 + 按模型执行的对话/步骤 + 复盘
- 让读者看到模型"落地后"长什么样

### 6 · 模型背后的系统逻辑

⭐ 最重要的章节：

- 模型的底层假设是什么？
- 它服务于什么世界观？
- 它与其他相邻模型（同领域的替代方案）如何对比？
- 画出"模型的哲学基础 SVG 图"

### 7 · 误读陷阱

读者常把模型用错的典型方式。

### 8 · 模型的盲点

- 模型处理不了的问题
- 模型的意识形态预设
- 模型的文化局限

## 密度要求

- 总字数：6000-12000（模型书比论证书略短）
- 模型步骤详解：每步 500-1000 字
- 实操剧本：≥ 2 个，每个 400+ 字
- 适用/失效矩阵：≥ 9 个格子
- SVG 图：≥ 3 张

## 禁忌

- 不要把书缩成"四步法总结" → 还要剖析每一步
- 不要只给成功剧本 → 必须有失效情境
- 不要把模型讲得像神药 → 暴露边界是诚实的表现
