# 事实系统化四步法

这是 Book Distiller 最核心的方法论。没有这一步，产出就回到"事实堆砌+观点罗列"的旧路。

## 为什么事实需要系统化

直接拿"好引用"去支撑观点，是初级做法。原因：

- 你**挑选**了事实 → 读者不知道你漏掉了什么
- 事实之间**无结构** → 读者看不到模式
- 结论容易**以偏概全** → 3 个锚点就敢下结论

系统化的事实则相反：

- **穷尽**相关事实 → 读者知道你看完了全貌
- **结构化组织** → 模式自己浮现
- **交织显现** → 系统逻辑无需额外论证就被看见

**"系统化的事实本身就是最强的论证"** —— 这是本方法论的核心信念。

---

## 步骤 1 · 穷尽式提取

### 原则

围绕母题，**不挑只收**。把书里所有相关事实拉出来。

### 五类事实全要

| 类型 | 说明 | 举例（金瓶梅） |
|---|---|---|
| 情节性事实 | 谁做了什么 | 西门庆第 27 回在葡萄架下折磨潘金莲 |
| 言论性事实 | 谁说了什么（原话） | 西门庆："咱闻那佛祖西天……" |
| 数据性事实 | 数字/时间/频率 | 西门庆家银子被单独提到 47 次 |
| 结构性事实 | 章节编排/视角切换 | 全书 100 回，西门庆死在第 79 回（剩 21 回写"他死后") |
| 沉默性事实 | 作者没写的 | 西门庆从未出现过父亲 |

**沉默性事实最容易被忽略，但往往最关键**——它揭示作者的价值排序、时代的禁忌、人物的盲区。

### 做法

1. 先列 2-4 个维度（见步骤 2），每个维度下独立穷尽
2. 用表格或列表记录，每条带章节定位
3. **不急着判断重要性**——先收集再筛选
4. 粗略估量数量：人物驱动书至少 30-50 条，论证型至少 20-40 条

### 止境

"我感觉还有没找完"时就还没完。直到你能说"再找也是重复"，才算穷尽。

---

## 步骤 2 · 维度分类（定坐标系）

### 原则

杂乱的事实必须进坐标系。坐标系不是随意挑的，**由基因决定**。

### 基因 × 坐标系表

| 主基因 | 常用坐标系 | 实例 |
|---|---|---|
| 人物 | 关系域 × 时间阶段 | 金瓶梅：情/钱/权/家庭/社交 × 前/中/后 |
| 叙事 | 情节弧 × 视角 | 百年孤独：起/承/转/合 × 布恩迪亚家族代际 |
| 论证 | 命题 × 证据类型 | 乌合之众：主/子命题 × 史例/心理实验/个人观察 |
| 模型 | 步骤 × 适用边界 | 非暴力沟通：观察/感受/需要/请求 × 成功/失败情境 |
| 史料 | 时间 × 地域 × 行动主体 | 人类简史：7 万年尺度 × 多区域 × 智人/他种 |
| 美学 | 意象 × 修辞 × 文化坐标 | 瓦尔登湖：自然意象 × 隐喻/直陈 × 超验主义 |
| 体验 | 感知类型 × 层次深度 | 冥想类书：身/心/境 × 初/中/深 |

### 做法

1. 看本书主基因 → 选坐标系
2. 画一个二维表：行是维度 A 的分类，列是维度 B 的分类
3. 把步骤 1 的事实填进格子
4. 空格子要**主动警觉**：是因为作者没写？还是我没找？

### 实战示例（金瓶梅）

```
          前 30 回           中 40 回           后 30 回
情域     -----------------  -----------------  -----------------
          占有欲强烈         消费欲主导         麻木+猎奇
          事实: #3.1,#6.2    事实: #27,#33      事实: #72,#75
          ...

钱域     -----------------  -----------------  -----------------
          原始积累           扩张投资           暴富中空转
          事实: #7.3,#9.1    事实: #30,#45      事实: #65,#69
          ...

权域     -----------------  -----------------  -----------------
          买通地方小吏       结交蔡太师集团     提督荫官
          事实: #10,#17      事实: #30,#55      事实: #70,#76
          ...

家庭域   -----------------  -----------------  -----------------
          妻妾配置完成       内斗激烈化         李瓶儿/官哥死
          ...               ...                ...

社交域   -----------------  -----------------  -----------------
          酒肉结拜兄弟       豪绅交际           群商趋附
          ...               ...                ...
```

填完这个矩阵，产出的"事实矩阵 SVG"就有了骨架。

## 步骤 3 · 模式识别（找单维规律）

### 原则

每个维度（表格的一行或一列）内部，**找规律**。

规律类型：
- **递增/递减**：程度单调变化
- **循环**：重复出现的周期
- **阶段跳变**：有明显分水岭
- **分岔**：同起点不同走向
- **镜像/对称**：对应关系

### 规则

每个规律必须有 **≥ 3 个事实锚点**支撑。2 个算巧合，3 个才算规律。

### 做法

1. 看一行（或一列）的事实序列
2. 问："这些事实有什么共同的演化逻辑？"
3. 尝试概括，回检是否真的覆盖所有事实
4. 如果有反例，调整规律描述或标记"局部规律"

### 实战示例

**金瓶梅 · 情域 → 时间轴**：
- 前期：对金莲、瓶儿等"占有式"猛攻（事实 #3/#6/#17）
- 中期：床笫器物化、"玉箫/春梅"成为消费品（#27/#33/#38）
- 后期：对宋惠莲、王六儿、如意儿——**越来越不在乎对象是谁**（#72/#75/#79）
- **规律：情欲从占有 → 消费 → 麻木**——典型的边际效用递减

这个规律直接变成机制层的材料。

## 步骤 4 · 跨维度交织（看系统浮现）

### 原则

**孤立规律还不是系统**。系统在**多个维度交织**中浮现。

### 做法

1. 列出各维度的规律
2. 问：这些规律之间有互相转化/互相依赖/互相放大/互相抵消吗？
3. 画出交织图——箭头、循环、三角、层级

### 交织形态

- **三角循环**：A → B → C → A（闭环自强化）
- **链条**：A → B → C（单向驱动）
- **双螺旋**：A 和 B 互相作为因果，螺旋上升/下坠
- **金字塔**：底层支撑上层
- **对立拉锯**：A 推，B 拉，平衡移动

### 实战示例

**金瓶梅 · 跨维度交织**：

```
       情欲域           钱欲域
         │                │
         ▼                ▼
  女色消费 ──→ 钱(挥霍+炫耀) ──→ 钱(更刚性的追逐)
         ▲                │
         │                ▼
  权力保护伞 ◀── 行贿买官 ◀──┘
         │
         ▼
  更多情欲空间
```

交织浮现的系统：**情欲→消费→钱荒→行贿→权力→情欲空间→更多情欲**，闭环自噬，最终自毙。

这就是"欲望的自噬循环"——书的系统逻辑。**它是从事实系统中浮现的，不是作者或你强加的**。

---

## 可视化（辅助）

每本书至少产出三张图：

1. **全貌地图**（SVG）——本书系统逻辑的总览
2. **事实矩阵**（SVG 表格）——步骤 2 的坐标系
3. **交织图**（SVG 节点/箭头）——步骤 4 的跨维度关系

水墨风，线条简洁，信息密度高不炫技。内联 SVG，无 CDN 依赖。

## 常见失败模式

### 失败 1 · 偷懒取样

只找"支持我观点"的事实。**反制**：强迫自己找 3 条反例，证明规律依然成立（或调整规律）。

### 失败 2 · 维度硬凑

看人物硬要按"情/钱/权"分。对有的书不适用。**反制**：维度是从事实中浮现的，不是预设的。不合就换。

### 失败 3 · 规律虚飘

"他越来越孤独"——模糊规律。**反制**：规律必须能转成可验证的描述。"从第 X 回开始，他独白占比从 Y% 升到 Z%"——这才算。

### 失败 4 · 交织强行拉郎

维度之间没关系硬说有。**反制**：交织的箭头必须有事实锚点支撑，不能纯推测。

---

## 检查清单

完成事实系统化后，检查：

- [ ] 至少 20 个事实被处理（人物/叙事书 30+）
- [ ] 至少 2 个维度的坐标系明确
- [ ] 每个规律有 ≥ 3 事实锚点
- [ ] 跨维度交织有显式图示
- [ ] 有反例检查（找不到反例也要标注"尝试过"）
- [ ] 所有事实带章节定位或原文引用

全部过关才能进步骤 6（骨架组装）。
